# BinAgg 再現追試レポート: DP 線形回帰と統計的保証付き合成データ生成

**対象 (Issue #16)** — Lin, Slavković & Bhoomireddy (2026),
*Differentially Private Linear Regression and Synthetic Data Generation with
Statistical Guarantees*, AISTATS 2026 [1]。
実装: [Shuronglin/BinAgg](https://github.com/Shuronglin/BinAgg) @ `13c09bb` (2026-05-27) [2]。

実行環境: Python 3.12.12 / numpy 2.4.6 / scipy 1.17.1 / pandas 3.0.3（`requirements.txt`）。
全乱数はシード固定。本レポートの数値は `results/` の生成物に対応する。

## Abstract

差分プライバシー（個々のレコードの有無が出力からほとんど判別できないことを保証する枠組み）
下での線形回帰について、論文 [1] が提案する手法（BinAgg）の公式実装 [2] を固定環境で動かし、
論文の中核的な主張を追試した。**結論として、主張はおおむね再現された。** すなわち、
(i) この手法が与える信頼区間はほぼ名目どおりの被覆を達成し、ノイズを考慮しない素朴な区間が
被覆を大きく失うのと対照的であること、(ii) 合成データに素朴に回帰すると係数が偏る一方、
この手法のバイアス補正がそれを回復すること、(iii) 実データでもプライバシーの強さに応じた
精度のトレードオフが期待どおり観測されること、を確認した（詳細は §4）。

残課題の方向性としては、(a) 評価データ（UCI Air Quality）に潜む欠損値表現の前処理を正す、
(b) 論文本文の図表との定量的な突き合わせ、(c) 設定（データ生成過程・標本数・実データ）を
広げた感度分析、が挙げられる（詳細は §6）。

## 1. 用語と記法

専門外の読者（学部生・大学院生）でも本文を追えるよう、主要な用語を先に定義する。
正確な定義・定理は論文 [1] を参照。

- **差分プライバシー (DP, Differential Privacy)**: データセットからレコードを 1 件
  追加・削除しても、解析結果（出力）の確率分布がほとんど変わらないことを保証する枠組み。
  「ある個人が含まれているか」を出力から見分けにくくする。
- **μ-GDP (Gaussian Differential Privacy)**: DP の強さを 1 つの実数 $\mu$ で測る定式化。
  $\mu$ が小さいほど加えるノイズが大きく、プライバシー保護は強い（精度は下がる）。本稿では
  「プライバシー予算」と呼ぶ。$(\varepsilon,\delta)$-DP へ換算する関数も実装に含まれる [2]。
- **PrivTree**: データ空間を再帰的に分割し、DP を満たすヒストグラム的な区画（**bin**）を作る
  既存アルゴリズム。各 bin で件数と和を集計する。
- **集約量 (aggregates)**: 各 bin の件数 $\mathrm{count}$、特徴量の和 $\mathrm{sum}(X)$、
  目的変数の和 $\mathrm{sum}(y)$。BinAgg はこれらにノイズを加えて公開・解析に用いる。
- **感度 (sensitivity)**: レコードを 1 件変えたときに集約量が動きうる最大量。
  加えるノイズの大きさを決める基準になる。
- **OLS (最小二乗法, Ordinary Least Squares)**: ノイズを加えない通常の線形回帰。
  本稿では非プライベートな参照値として用いる。
- **被覆率 (coverage)**: 同じ手続きを多数回繰り返したとき、構成した 95% 信頼区間 (CI) が
  真の係数を含んだ割合。手続きが妥当なら名目水準 $0.95$ に近づく。
- **「naive（素朴）」の 2 つの用法**（混同しやすいので明記）:
  (a) **naive 推定量** — バイアス補正項を引かず、ノイズの分散も無視した素朴な点推定・区間。
  (b) **合成データへの素朴 OLS** — 生成された合成データにそのまま通常の OLS を当てたもの。
- **TVD (全変動距離, Total Variation Distance)**: 2 つの確率分布の隔たりを測る指標。
  合成データ評価では「周辺分布の一致」を測るのに使われがちだが、本手法はこれを主指標にしない。

記法（チルダ $\tilde{\cdot}$ は「ノイズを加えた DP 版」を表す）:

| 記号 | 意味 |
|---|---|
| $\mu$ | プライバシー予算（μ-GDP、小さいほど強い） |
| $\beta,\ \tilde\beta$ | 真の回帰係数 / DP バイアス補正済み推定 |
| $S,\ \tilde S$ | 特徴量の bin 内和（真 / ノイズ付き） |
| $t,\ \tilde t$ | 目的変数の bin 内和（真 / ノイズ付き） |
| $W,\ \tilde W$ | bin 件数の逆数を対角に並べた重み行列 |
| $\tilde D$ | バイアス補正行列（Thm 4.2） |
| $\tilde\Sigma$ | 係数推定の分散共分散行列（サンドイッチ推定） |
| $n,\ d,\ K$ | 標本数 / 特徴次元 / bin 数 |

> 事実と推定の区別: [n] 付きは論文・コードに基づく事実。タグ無しの「考察」は本追試の解釈。

## 2. 背景と手法 (出典: [1][2])

- **問題** [1]: DP 線形回帰では集約量にノイズを足すため、ノイズ入りの二乗和
  $\tilde S^\top \tilde W \tilde S$ が真の二乗和 $S^\top W S$ より系統的に大きくなり、点推定が
  $0$ 方向へ縮小して偏る。さらにノイズの分散を無視した標準誤差は過小評価となり、信頼区間が
  狭くなりすぎて被覆を失う。**点推定のバイアス**と**分散推定の過小評価**という二つの歪みが生じる。
- **BinAgg の流れ** [1][2]:
  1. **PrivTree** でデータ空間を private に分割する（binning）。
  2. 各 bin の $\mathrm{count}/\mathrm{sum}(X)/\mathrm{sum}(y)$ に Gaussian ノイズを加える
     （μ-GDP）。既定の予算配分は $(\text{binning},\text{count},\text{sum}_x,\text{sum}_y)=(1,3,3,3)$、
     すなわち分割に 10%、各集約に 30% ずつを割り当てる。
  3. **Algorithm 2（DP 線形回帰）**: 重み付き最小二乗にバイアス補正項 $\tilde D$ を入れて
     点推定 $\tilde\beta$ を求める（下式）。補正項 $\tilde D$ は、ノイズが二乗和に持ち込む
     系統的な増分を差し引く役割をもつ。分散は**サンドイッチ推定量**（中心の行列 $\tilde H$ を
     両側から $\tilde M^{-1}$ で挟む形）$\tilde\Sigma$ で求め、これから信頼区間を構成する（Thm 4.2）。
  4. **Algorithm 3（合成データ生成）**: 同じ集約から bin 単位で合成標本を生成する。回帰と合成は
     同じノイズ実現を共有でき（Corollary 3.1）、**同一のプライバシー予算**で両方を出力できる。

点推定とその分散は次式で与えられる（記号は §1。$\tilde M = \tfrac1K\tilde S^\top\tilde W\tilde S-\tilde D$）:

$$\tilde\beta = \bigl(\tilde S^\top \tilde W \tilde S - \tilde D\bigr)^{-1}\tilde S^\top \tilde W \tilde t,
\qquad \tilde\Sigma = \tilde M^{-1}\,\tilde H\,\tilde M^{-1}.$$

- **評価方針** [1]: 合成データの良否を「周辺分布の保存（例: TVD）」ではなく、**回帰分析の保証
  （点推定の正しさと信頼区間の妥当性）**で測る点が本手法の特徴である。本追試もこの方針に従う。

## 3. 実験設定

指標・ベースライン・予算・データの詳細は `docs/plans/experiment-plan.md`。要点を以下に示す。

- **主指標**:
  - **95% CI 被覆率** — 多数試行のうち CI が真値を含んだ割合
    $\frac{1}{T}\sum_{j}\mathbf{1}\{\beta_j \in \mathrm{CI}_j\}$。妥当なら $\approx 0.95$。
  - **係数バイアス** — $\mathbb{E}[\hat\beta]-\beta$ の絶対値の平均（補正あり $\tilde\beta$ と naive の対比）。
  - **相対 L2 係数誤差** — $\lVert\hat\beta-\beta\rVert_2 / \lVert\beta\rVert_2$（小さいほど良い）。
  - **CI 幅** — 区間の上限と下限の差の平均。
  - 補助指標: **モーメント誤差** — 合成データと元データの平均・標準偏差の差の絶対値。
  - 周辺分布の TVD ではなく**回帰の保証**を主軸に置く [1]。
- **ベースライン**: 非プライベート OLS（有用性 (utility) の参照値・被覆の比較対象）、
  naive 推定量（補正なし）、合成データへの素朴 OLS（§1 の 2 用法を参照）。
- **プライバシー予算**: μ-GDP で $\mu \in \{0.5,\,1.0,\,2.0,\,5.0\}$（小→強い保護、大→弱い保護）。
- **データ**:
  - シミュレーション: $X\sim\mathrm{Uniform}(0,10)^3$、$\beta=[1.5,-2,0.5]$、
    $y=X\beta+\varepsilon,\ \varepsilon\sim\mathcal{N}(0,1)$、$n=2000$。定義域の上下限 (bounds) は
    **既知の生成分布から**設定する（データ非依存。DP では bounds を観測データから決めると漏えいになる）。
  - 実データ: UCI Air Quality 同梱 CSV。目的変数 CO(GT)、特徴 12 次元。bounds は
    ドメイン知識から固定（BinAgg の example に準拠）。

## 4. 結果

### E1 — 統計的保証: 95% CI 被覆率（シミュレーション, $T=300$ 試行/$\mu$）

`results/e1_coverage.csv` / `figures/e1_coverage.png`

![E1: CI coverage and coefficient bias vs μ](results/figures/e1_coverage.png)

| μ | 被覆率 BinAgg(補正) | 被覆率 naive | 平均\|バイアス\| BinAgg | 平均\|バイアス\| naive | 平均 CI 幅 |
|---:|---:|---:|---:|---:|---:|
| 0.5 | **0.917** | 0.077 | 0.064 | 0.066 | 0.518 |
| 1.0 | **0.931** | 0.149 | 0.032 | 0.032 | 0.291 |
| 2.0 | **0.934** | 0.227 | 0.019 | 0.019 | 0.181 |
| 5.0 | **0.918** | 0.318 | 0.012 | 0.012 | 0.103 |

- **事実**: BinAgg の補正版 CI は全 $\mu$ で被覆率 0.92–0.93 と名目水準 0.95 に近い。係数別でも
  0.89–0.97 の範囲に収まる（$\beta_2=-2.0$ が僅かに低めの $\approx 0.89$）。
- **事実**: ノイズの分散を無視した naive 区間の被覆率は 0.08–0.32 と大きく不足する。
  CI 幅は $\mu$ が増えるにつれ単調に縮小する（0.52→0.10）。
- **考察**: 「妥当な信頼区間」という主張は再現された。点推定のバイアスは本設定（条件数の良い
  一様計画）では補正版と naive でほぼ同等であり、**両者を分けているのは点推定よりも分散推定
  （サンドイッチ推定による標準誤差）**だと解釈できる。被覆率が 0.95 を僅かに下回るのは、
  この信頼区間が漸近的（標本数が大きい極限で正当化される）であることに伴う有限標本の近似誤差と
  整合的である（§6）。

### E2 — 合成データの回帰有用性とバイアス補正の効果（シミュレーション, 20 シード平均）

`results/e2_synthetic_utility.csv` / `figures/e2_utility.png`

![E2: regression utility vs μ](results/figures/e2_utility.png)

相対 L2 係数誤差 $\lVert\hat\beta-\beta\rVert_2/\lVert\beta\rVert_2$（小さいほど良い）:

| μ | OLS(元データ, 参照) | naive OLS(合成) | BinAgg DP 回帰 ($\tilde\beta$) |
|---:|---:|---:|---:|
| 0.5 | 0.003 | 0.575 | **0.093** |
| 1.0 | 0.003 | 0.327 | **0.058** |
| 2.0 | 0.003 | 0.172 | **0.031** |
| 5.0 | 0.003 | 0.064 | **0.018** |

- **事実**: 合成データへ素朴に OLS を当てると係数誤差が大きい（$\mu=0.5$ で 0.58）。BinAgg の
  バイアス補正推定は同一データから誤差 0.09 と約 1 桁小さくなり、$\mu$ が大きくなるにつれて
  OLS(元データ) の 0.003 に漸近する。
- **考察**: 「合成データをそのまま回帰すると偏り、補正によって真値を回復できる」という主眼を
  確認した。有用性を**回帰の保証**で測る論文の評価軸は、周辺分布の一致を見るより厳しく、
  実用上も意味のある基準であることを示している。

### E3 — 実データ再現: UCI Air Quality（$n=9357$, $d=12$, 5 シード平均）

`results/e3_airquality.csv` / `figures/e3_airquality.png`。実装の Option A/B/C をいずれも実行した
（A: 回帰のみ、B: 回帰＋合成で予算共有、C: 合成のみ）。

![E3: CI width and synthetic moment error vs μ](results/figures/e3_airquality.png)

| μ | 平均 95% CI 幅 | \|mean(y) 誤差\| | \|std(y) 誤差\| |
|---:|---:|---:|---:|
| 0.5 | 1.461 | 0.856 | 1.643 |
| 1.0 | 1.068 | 0.468 | 0.943 |
| 2.0 | 0.776 | 0.217 | 0.383 |
| 5.0 | 0.387 | 0.059 | 0.036 |

（モーメント誤差は、クリップ後の元データ mean=1.766 / std=1.554 を基準とした。理由は §6。）

- **事実**: CI 幅・合成データのモーメント誤差はいずれも $\mu$ が増えるにつれ単調に縮小する。
  Option C では合成標本数 9357（元データと同数。`preserve_sample_size` による）・使用 bin 数 159
  を確認した。Option A/B は同一予算で問題なく動作した。
- **考察**: 実データでもプライバシーと精度のトレードオフが定性的に期待と整合して観測され、
  実装は安定に動作する。一方、非プライベート OLS との係数の一致度は、本データでは妥当な
  有用性指標にならない（理由は §6）。

## 6. 限界と注意点

1. **欠損値センチネル `-200`（再現上の落とし穴・本追試の発見）**: UCI Air Quality は欠損を
   `-200` で表す。BinAgg 同梱 example のクリーニングはこれを除去しておらず、CO(GT) では
   1683/9357 行が `-200` である。生の $y$ は平均 $-34.21$ / 標準偏差 $77.65$ と汚染される。
   DP の経路は bounds $(0,15)$ へのクリップで一部遮蔽されるが、**(a) 合成データのモーメントを
   生の $y$ と比較すると無意味な巨大誤差になり**、**(b) 非プライベート OLS の参照自体が汚染**
   される（例: AH 係数 $-0.895$）。本追試は (a) をクリップ後基準で回避したが、E3 における
   DP–OLS の係数一致は有用性指標として信頼できない。**正しい前処理は `-200` 行の除去**であり、
   ベンチマーク設計上の重要点として記録する。
2. **単一の生成過程・単一の実データ**: E1/E2 は 1 種類の線形生成過程、E3 は 1 データセットのみ
   に基づく。一般化には、設定（特徴分布・次元・条件数・誤差分布）と複数データへの拡張が必要。
3. **漸近信頼区間の有限標本近似**: 被覆率が 0.95 を僅かに下回る（0.92–0.93）のは、漸近正規
   近似に伴う有限標本の誤差と整合的である。$n$ や bin 数を増やした感度分析は未実施。
4. **論文本文との数値対応は未照合**: 本追試は実装 [2] の挙動の再現であり、論文 [1] の図表との
   定量的な一致は確認していない（arXiv 本文の精読は Issue タスクの残作業）。
5. **bounds の与え方**: シミュレーションは生成分布の既知範囲、実データはドメイン知識で固定した。
   bounds を private に推定する経路（予算の一部を割り当てる方法）は評価していない。

## 7. 結論

公式実装 BinAgg を固定環境で動かし、論文 [1] の中核的な主張を再現できた。すなわち、
(i) バイアス補正とサンドイッチ分散推定による **95% 信頼区間はほぼ名目どおりの被覆を達成**し、
ノイズを無視する素朴な推定は被覆を失う (E1)、(ii) **合成データ上の回帰は補正なしでは偏り、
BinAgg の補正が真値を回復する** (E2)、(iii) 実データでもプライバシーの強さに応じた精度の
トレードオフが観測される (E3)。「合成データの有用性を回帰の統計的保証で測る」という論文の
評価軸の有効性を、被覆率という直接的な指標で確認した点が本追試の主な貢献である。残課題は
§6 の各項（特に `-200` の前処理の是正、論文数値との照合、設定を広げた感度分析）である。
